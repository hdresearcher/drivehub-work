# Security Policy

## Reporting a Vulnerability

Onroute Express Inc. takes the security of its software and services
seriously. If you believe you have found a security vulnerability in this
repository or in any deployed application derived from it, we ask that you
report it to us privately, responsibly, and in good faith.

**Please do not open a public issue, pull request, or discussion for
security matters.**

### How to report

Send a report to: **eugene@onrouteexpress.com**

For sensitive disclosures, encrypt with PGP:

- **Fingerprint:** 991B3FBDCB0AFA906ECEE5C17F9C26B3BE3D9A5B
- **Public key:** https://keys.openpgp.org/vks/v1/by-fingerprint/991B3FBDCB0AFA906ECEE5C17F9C26B3BE3D9A5B

Please include, to the extent possible:

- A description of the vulnerability and its potential impact
- Steps to reproduce, including proof-of-concept code or payloads
- The affected component, commit SHA, deployed URL, or version
- Any relevant logs, screenshots, or recordings
- Your name or handle if you wish to be credited

## Scope

In scope:

- This repository and its deployed artifacts
- `drivehub.work` and subdomains
- `onroute.express` and subdomains
- Associated API endpoints and integrations

Out of scope:

- Denial-of-service testing against production systems
- Social engineering of Onroute Express Inc. employees or contractors
- Physical attacks
- Automated scanning that generates excessive traffic
- Vulnerabilities in third-party services we do not operate

## Our commitment

- We will acknowledge receipt of your report within **3 business days**.
- We will provide an initial assessment within **10 business days**.
- We will keep you informed of remediation progress.
- We will credit you in a public advisory once the issue is resolved, if
  you wish.
- We will not pursue legal action against researchers who act in good faith
  and in accordance with this policy.

## Safe Harbor

Activity conducted in a manner consistent with this policy is considered
authorized. We will not initiate legal action against, or support action by
others against, researchers for good-faith security research.

## Supported Versions

Only the latest release and `main` branch receive security updates. Older
branches, tags, and preview deployments are provided as-is.
